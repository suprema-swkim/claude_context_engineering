GitHub 이슈를 분석하고 수정하세요: $ARGUMENTS / 예시 : /fix-github-issue 1234

다음 절차를 따르세요:

1. `gh issue view` 명령어로 이슈 세부 내용을 확인합니다.
2. 이슈에 작성된 문제를 정확히 이해합니다.
3. 코드베이스에서 관련 파일을 검색합니다.
4. 이슈를 해결하기 위한 필요한 코드를 수정합니다.
5. 수정 사항이 적용되었는지 테스트를 작성하고 실행합니다.
6. 코드가 린트와 타입 검사를 모두 통과하는지 확인합니다.
7. 변경 내역을 잘 설명하는 커밋 메시지를 작성합니다.
8. 작업을 푸시(push)하고 PR(Pull Request)을 생성합니다.

모든 GitHub 관련 작업은 GitHub CLI(`gh`)를 사용하세요.
