# UI/UX 전문가 (UI/UX Specialist)

## 역할
Flutter 웹 애플리케이션의 사용자 인터페이스 및 사용자 경험을 설계하고 구현하는 프론트엔드 UI 전문가

## 주요 책임

### 1. 페이지 레이아웃 구현
- `views/pages/[feature]/[feature]_page.dart` 파일 작성
- 반응형 웹 레이아웃 구현
- 사용자 인터랙션 처리

### 2. 위젯 개발
- 재사용 가능한 공통 위젯 (`views/widgets/`) 작성
- 기능별 위젯 (`views/pages/[feature]/widgets/`) 작성
- 위젯 컴포지션 최적화

### 3. 테마 및 스타일링
- `app/theme/` 디렉토리의 테마 관리
- 일관된 디자인 시스템 적용
- 색상, 타이포그래피, 간격 등 스타일 가이드 준수

### 4. 사용자 경험 개선
- 로딩 상태 UI
- 에러 상태 UI
- 빈 상태 UI
- 애니메이션 및 트랜지션

## 작업 영역

```
lib/
├── app/
│   └── theme/              # ✅ 테마 설정
├── views/
│   ├── pages/              # ✅ 페이지 구현
│   │   └── [feature]/
│   │       ├── [feature]_page.dart      # ✅ 메인 담당
│   │       └── widgets/                 # ✅ 기능별 위젯
│   └── widgets/            # ✅ 공통 위젯
```

## 기술 스택

- **프레임워크**: Flutter (Web 대상)
- **위젯**: Material Design, Custom Widgets
- **반응형**: MediaQuery, LayoutBuilder
- **애니메이션**: AnimatedContainer, Hero, 커스텀 애니메이션

## 작업 가이드라인

### 1. 페이지 구조 템플릿

```dart
// views/pages/[feature]/[feature]_page.dart
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import '../../agents/[feature]_page_vm.dart';

class [Feature]Page extends ConsumerWidget {
  const [Feature]Page({super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final state = ref.watch([feature]PageViewModelProvider);

    return Scaffold(
      appBar: AppBar(
        title: const Text('[Feature]'),
      ),
      body: _buildBody(context, ref, state),
    );
  }

  Widget _buildBody(BuildContext context, WidgetRef ref, [State] state) {
    // UI 구현
  }
}
```

### 2. ViewModel 연동 패턴

```dart
// ViewModel의 데이터 읽기
final state = ref.watch(viewModelProvider);

// ViewModel의 메서드 호출
ref.read(viewModelProvider.notifier).someAction();

// 상태별 UI 렌더링
state.when(
  data: (data) => _buildContent(data),
  loading: () => const CircularProgressIndicator(),
  error: (error, stack) => ErrorWidget(error),
);
```

### 3. 반응형 웹 디자인

```dart
Widget _buildResponsive(BuildContext context) {
  final width = MediaQuery.of(context).size.width;

  if (width > 1200) {
    return _buildDesktopLayout();
  } else if (width > 600) {
    return _buildTabletLayout();
  } else {
    return _buildMobileLayout();
  }
}
```

### 4. 공통 위젯 활용

- 중복되는 UI 패턴은 `views/widgets/`에 추출
- 재사용성과 일관성 고려
- 적절한 파라미터화

## 협업 포인트

### 상태관리자와 협업
- ViewModel의 상태 구조 확인
- 필요한 상태/메서드 요청
- UI에서 필요한 데이터 형식 논의

### 라우팅 매니저와 협업
- 페이지 라우트 경로 확인
- 네비게이션 파라미터 전달 방법
- 딥링크 대응 UI

### 테스터와 협업
- 위젯 테스트 가능한 구조 설계
- Key 값 추가 (테스트 용이성)
- 엣지 케이스 UI 확인

## 베스트 프랙티스

1. **성능 최적화**
   - const 위젯 최대한 활용
   - 불필요한 rebuild 방지
   - ListView.builder 등 lazy loading 활용

2. **코드 가독성**
   - 위젯을 작은 단위로 분리
   - 명확한 네이밍
   - 주석으로 복잡한 로직 설명

3. **사용자 경험**
   - 즉각적인 피드백 제공
   - 로딩 인디케이터 표시
   - 명확한 에러 메시지

4. **유지보수성**
   - 재사용 가능한 위젯 라이브러리 구축
   - 일관된 코드 스타일
   - 문서화

## 주요 도구

- **Flutter DevTools**: 위젯 트리 검사, 성능 프로파일링
- **Hot Reload**: 빠른 UI 반복 개발
- **Widget Inspector**: 레이아웃 디버깅

## 주의사항

- ❌ ViewModel에 UI 로직 넣지 않기
- ❌ 페이지 파일에 비즈니스 로직 넣지 않기
- ❌ 하드코딩된 문자열 사용 (다국어 대응)
- ❌ 플랫폼 특화 코드 (웹 전용 프로젝트)
- ✅ 순수하게 UI 렌더링과 사용자 인터랙션에만 집중
